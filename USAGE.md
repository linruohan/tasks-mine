# Tasks Mine 使用指南

## 快速开始

### 1. 编译运行

```bash
# 开发模式运行
cargo run

# 发布版本编译
cargo build --release
```

### 2. 界面导航

应用启动后，左侧为导航栏，包含以下模块：

- **CodeHub**: MR 管理
- **DTS**: 问题单管理
- **Excel**: Excel 数据处理
- **Hive**: Hive 平台管理
- **Requirement**: 需求管理

## 功能详解

### CodeHub - MR 管理

**主要功能：**
- 添加 MR：点击"添加 MR"按钮，填写 MR 信息
- 查询 MR：使用日期范围筛选器查询指定周期的 MR
- 统计工作量：自动计算新增行数、删除行数和净变化

**操作步骤：**
1. 设置开始日期和结束日期
2. 点击"查询"按钮
3. 查看统计卡片显示的工作量数据
4. 在列表中查看详细的 MR 信息

### DTS - 问题单管理

**主要功能：**
- 添加问题单：记录新发现的问题
- 查询问题单：搜索和筛选问题单
- 统计回归率：自动计算问题单的提交和回归数量

**问题单状态：**
- 提交：已提交待处理
- 回归：已验证通过
- 其他：其他自定义状态

### Excel - 数据处理

**主要功能：**
- 读取 Excel：支持 .xlsx 和 .xls 格式
- 写入 Excel：导出数据为 .xlsx 格式
- 批量处理：批量导入导出数据

**快速操作：**
1. 点击"选择文件读取"导入 Excel 文件
2. 数据将在预览区显示
3. 点击"导出数据"可将当前数据导出

**模板下载：**
- MR 模板：用于批量导入 MR 数据
- DTS 模板：用于批量导入问题单数据

### Hive - 平台管理

包含三个子功能模块：

#### 1. 失败用例分析
- 搜索失败用例
- 查看错误信息
- 导出分析报告

#### 2. 虚拟环境管理
- 占用环境：标记环境为占用状态
- 释放环境：释放已占用的环境
- 部署环境：执行环境部署操作

**环境状态：**
- 🟢 空闲：可以占用
- 🟡 占用：正在使用中
- 🔵 部署中：正在部署

#### 3. 工程 Job 管理
- 拉起 Job：启动新的工程 Job
- 续跑 Job：继续执行暂停的 Job
- 失败分析：组织和查看失败分析报告

### Requirement - 需求管理

**主要功能：**
- 添加需求：创建新的需求记录
- 需求版本管理：跟踪需求的版本信息
- 测试周期管理：设置测试开始和结束时间
- 状态监控：监控需求的进行状态

**需求状态：**
- 🔵 进行中：正在测试中
- 🟢 已完成：测试完成
- 🔴 延期：超过截止日期

## 数据持久化

当前版本的数据存储在内存中，重启应用后数据会丢失。

**计划功能：**
- SQLite 数据库持久化
- 数据导入导出
- 配置文件管理

## HTTP 请求配置

Request 模块用于与外部系统集成：

**配置步骤：**
1. 在代码中配置 API 端点
2. 设置 Cookie 认证信息
3. 使用对应的 fetch 方法获取数据

**示例：**
```rust
let client = RequestClient::new();
// 登录获取 Cookie
client.login("https://api.example.com/login", "username", "password").await?;
// 获取数据
let mrs = client.fetch_merge_requests("https://api.example.com/mrs").await?;
```

## 故障排除

### 编译错误
确保使用最新的 Rust 工具链：
```bash
rustup update
```

### 组件库路径错误
确保 gpui-component 在正确的相对路径：
```bash
d:\codehub\gpui-component\
```

### 运行时错误
查看控制台输出的错误日志，使用 `RUST_LOG=debug` 启用详细日志：
```bash
RUST_LOG=debug cargo run
```
